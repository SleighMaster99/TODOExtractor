# Project Brief: Todo Extractor

**작성 상태:** 진행 중 (Draft)
**마지막 업데이트:** 2025-11-20

---

## Executive Summary

**Todo Extractor**는 Visual Studio 2022 확장 프로그램으로, 개발자가 코드에 작성한 TODO 주석을 자동으로 추출하여 GitHub/GitLab 이슈로 변환하는 도구입니다.

**핵심 문제:** 개발자들은 코딩 중 떠오르는 작업을 `//TODO:` 형태로 주석에 남기지만, 이를 체계적으로 추적하고 이슈화하는 과정이 수동적이고 번거롭습니다.

**대상 시장:** Visual Studio 2022를 사용하는 개발자 및 개발팀

**핵심 가치:** 사용자 정의 마커 기반 주석 추출과 GitHub/GitLab 자동 이슈 생성을 통해 코드 내 작업 항목을 효율적으로 관리하고 추적할 수 있습니다.

---

## Problem Statement

**현재 상태와 문제점:**

개발자들은 코딩 중 다음과 같은 작업을 주석으로 남깁니다:
- 추후 구현해야 할 기능
- 리팩토링이 필요한 부분
- 버그 수정 사항
- 성능 개선 항목
- 문서화가 필요한 영역

하지만 이러한 주석들은:
- 코드베이스 전체에 흩어져 있어 관리가 어렵습니다
- 수동으로 찾아서 이슈 트래커에 등록해야 하므로 시간이 소요됩니다
- 개발 흐름을 방해하지 않으려면 나중으로 미루게 되고, 결국 잊혀집니다
- 팀 차원에서 우선순위를 관리하거나 할당하기 어렵습니다

**문제의 영향:**

- **생산성 저하:** 개발자가 주석을 수동으로 찾고 이슈로 옮기는 데 시간을 낭비합니다
- **작업 누락:** 중요한 작업이 코드 주석에 묻혀 실행되지 않습니다
- **추적 불가:** 남겨진 작업의 전체 현황과 우선순위를 파악할 수 없습니다
- **협업 어려움:** 팀원 간 작업 항목 공유와 할당이 비효율적입니다

**기존 솔루션의 한계:**

- **수동 검색:** VS의 "Find in Files"로 TODO를 찾아도 이슈화는 별도 작업
- **기존 확장 프로그램:** Task List 등은 단순 나열만 제공하며 이슈 관리 시스템과 연동 부족
- **외부 스크립트:** grep 등 CLI 도구는 VS 환경과 통합되지 않아 개발 흐름 단절

**해결의 시급성:**

현대 애자일 개발에서 작업의 가시성과 추적성은 필수입니다. 코드 내 작업 항목을 즉시 이슈 트래커로 전환할 수 있어야 진정한 지속적 개선이 가능합니다.

---

## Proposed Solution

**핵심 컨셉:**

Todo Extractor는 Visual Studio 2022에 긴밀하게 통합된 확장 프로그램으로, 개발자의 코드 내 작업 주석을 자동으로 발견하고 이슈 관리 시스템으로 변환하는 솔루션입니다.

**작동 방식:**
1. 사용자가 커스텀 마커(예: `TODO:`, `FIXME:`, `OPTIMIZE:`)를 정의
2. 확장 프로그램이 지정된 경로의 코드를 스캔하여 `//마커` 패턴의 주석 추출
3. 추출된 항목을 그룹화, 필터링, 우선순위화
4. GitHub/GitLab API를 통해 자동으로 이슈 생성 또는 로컬 파일로 저장

**기존 솔루션 대비 차별화 요소:**

- **완전한 커스터마이제이션:** 사용자 정의 마커, 중요도 설정, 다중 마커 지원
- **양방향 출력:** 이슈 트래커 직접 연동과 로컬 파일 저장(MD, CSV, JSON, HTML, DB) 모두 지원
- **Visual Studio 네이티브 통합:** IDE 워크플로우를 벗어나지 않고 사용 가능한 세련된 UI
- **스마트 필터링:** 파일 확장자, 디렉터리 제외, 경로 지정 등 정교한 스캔 제어
- **미리보기 기능:** 이슈 생성 전 검토 및 선택적 실행 가능

**성공 요인:**

- **개발자 워크플로우 존중:** 코딩 흐름을 방해하지 않고 자연스럽게 통합
- **즉각적인 가치:** 설치 후 바로 사용 가능한 직관적 인터페이스
- **유연성:** 개인 개발자부터 팀까지 다양한 사용 사례 지원
- **오픈 생태계:** 주요 이슈 트래커(GitHub, GitLab)와 표준 파일 형식 지원

**제품 비전:**

개발자가 "나중에 해야지"라고 생각하는 순간, 그 생각을 코드 주석으로 남기면 자동으로 추적 가능한 작업 항목이 되는 세상을 만듭니다. 코드와 작업 관리 사이의 마찰을 제거하여 진정한 지속적 개선 문화를 촉진합니다.
