### 기본 정보

| 항목 | 내용 |
|------|------|
| **프로젝트명** | Todo Extractor |
| **플랫폼** | Visual Studio 2022 확장 프로그램 |
| **대상 사용자** | VS 2022를 사용하는 개발자/사원 |
| **개발 언어** | C# (VS 확장 표준) |

### 핵심 가치

개발자가 코딩 작업 중 생각나거나 추후 필요한 작업을 주석으로 작성하고, Todo Extractor를 통해 추출하면 GitHub 또는 GitLab 이슈에 등록할 내용이 리스트로 정리되어 **향후 이슈를 관리하기 쉽습니다.**

### 핵심 기능

사용자가 설정한 마커로 정해진 규칙에 맞는 주석(`//마커`로 시작)을 리스트로 추출합니다.

## 기능 요구사항 정리

### 1. 마커 관련 설정

| 기능 | 설명 |
|------|------|
| **다중 마커 지원** | 여러 마커 등록 가능 (리스트 형태) |
| **마커 중복 방지** | 동일 마커 중복 등록 불가 (유효성 검사) |
| **중요도 설정** | 마커별 중요도 (높음/보통/낮음), 중요도는 중복 가능 |
| **UI** | 세련된 리스트 UI로 추가/삭제/편집 |

**마커 추출 규칙:**
- `//마커`로 시작하는 라인만 추출
- 예: 마커가 `TODO:`이면 `//TODO:`로 시작해야 함
- 단순히 `TODO:`가 포함된 것은 추출하지 않음

### 2. 필터링 설정

| 기능 | 설명 |
|------|------|
| **파일 확장자 필터** | `.cs`, `.cpp`, `.js` 등 선택 |
| **제외 디렉터리** | `bin`, `obj`, `node_modules` 등 |
| **검사할 파일 경로** | 리스트로 여러 경로 등록 |
| **제외 패턴** | 특정 파일명 패턴 제외 |

### 3. 출력 설정

| 기능 | 설명 |
|------|------|
| **출력 방식** | 이슈 자동 생성 / 로컬 파일 저장 / 둘 다 |
| **파일 포맷** | Markdown, CSV, JSON, HTML, DB |
| **그룹화 방식** | 마커별, 파일별, 우선순위별 |
| **정렬 순서** | 파일 경로, 라인 번호, 마커 유형 |

### 4. GitHub/GitLab 연동

| 기능 | 설명 |
|------|------|
| **저장소 URL** | 대상 저장소 주소 |
| **인증 토큰** | 보안 저장 |
| **기본 라벨** | 생성되는 이슈에 자동 추가 |
| **담당자 지정** | 기본 담당자 설정 |

### 5. UI/UX 요구사항

- 버튼, 리스트 등 사용자가 사용하기 쉬운 UI
- 결과 미리보기 (이슈 생성 전 확인)
- 일괄 선택/해제 체크박스
- 검색/필터 기능

## MVP 범위

### 포함
- 마커 기반 주석 추출 (`//마커` 규칙)

- 로컬 파일 저장 (MD, CSV, JSON, HTML, DB)

### 제외
